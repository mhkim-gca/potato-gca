# 감자마켓 (potato-gca) — 작업 규칙

## 이 프로젝트는

- **개발 연습용** 폴더입니다. 사용자는 비개발자이며, 코딩을 배우는 중입니다.
- 기술 스택: **Next.js**(웹사이트 틀) + **Supabase**(데이터베이스·로그인 담당).
- 배포 방식: **GitHub에 올리면(push) → 연결된 Vercel이 자동으로 인터넷에 배포**합니다.
  별도 배포 명령은 필요 없고, 깃헙에 올리는 것만으로 실제 사이트에 반영됩니다.

## 사용자와 대화하는 방식 (가장 중요)

- **쉬운 말로 설명한다.** 개발 용어가 나오면 비개발자가 이해할 수 있게 풀어서 설명한다.
  - 예: "커밋(commit)" → "지금까지 작업한 내용을 저장 지점으로 남기는 것",
    "배포(deploy)" → "만든 걸 실제 인터넷에 올려서 누구나 볼 수 있게 하는 것".
- **지금 무엇을 하고 있는지 자주 알려준다.** 작업을 시작할 때 / 단계가 바뀔 때 / 끝났을 때
  "지금 ○○를 하는 중이에요", "이건 끝났고 다음은 ○○예요" 식으로 진행 상황을 말해준다.
- 한국어로 대화한다.

## 작업하는 방식

- **사용자가 지시한 것만 단계적으로 수행한다.** 시키지 않은 작업을 미리 진행하거나,
  여러 단계를 한꺼번에 몰아서 처리하지 않는다. 한 단계씩 진행하고 확인을 받는다.
- **내가 할 수 있는 일은 최대한 직접 한다.** 파일 작성·수정, 코드 실행, 설정 등
  도구로 처리할 수 있는 건 사용자에게 떠넘기지 말고 직접 끝낸다.
- **꼭 사람이 직접 해야 하는 것만** 골라서, **쉽고 명확하게 단계별로 안내한다.**
  (예: 브라우저에서 버튼 클릭, 외부 서비스 로그인, 비밀번호·키 발급 등)
  - 어디를 클릭하는지, 무엇을 입력하는지 구체적으로 알려준다.

## 개발을 마쳤을 때

- **작업이 끝나면 항상 바로 확인할 수 있게 열어준다.**
  - 화면(웹페이지) 작업이면 개발 서버(미리보기)를 띄워서 결과를 직접 보여주거나,
    열어볼 주소(예: http://localhost:3000)를 알려준다.
  - 사용자에게 "직접 확인해보세요"라고 떠넘기지 말고, 내가 먼저 확인한 결과(스크린샷·로그 등)를
    보여준 뒤, 사용자가 바로 눌러 볼 수 있는 링크/주소를 함께 안내한다.
